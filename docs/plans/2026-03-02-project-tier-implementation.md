## Background: How the JSON Controls Work

Each control in a `.json` file has this structure:

```json
{
  "Type": "ControlInfo",
  "Name": "ControlName",
  "Template": { "Id": "http://microsoft.com/appmagic/...", "Version": "x.y.z", ... },
  "Index": 0,
  "PublishOrderIndex": <unique int across all Controls/*.json>,
  "VariantName": "",
  "LayoutName": "",
  "MetaDataIDKey": "",
  "PersistMetaDataIDKey": false,
  "IsFromScreenLayout": false,
  "StyleName": "defaultXxxStyle",
  "Parent": "ParentControlName",
  "IsDataControl": false,
  "AllowAccessToGlobals": true,
  "OptimizeForDevices": "Off",
  "IsGroupControl": false,
  "IsAutoGenerated": false,
  "Rules": [
    { "Property": "PropName", "Category": "Data|Design", "InvariantScript": "value", "RuleProviderType": "Unknown" }
  ],
  "ControlPropertyState": [ "PropName", ... ],
  "IsLocked": false,
  "ControlUniqueId": "<unique int string across entire app>",
  "Children": []
}
```

**App-wide maximums (at time of writing):**
- `PublishOrderIndex`: max = 258 → use 259, 260, 261, ... for new controls
- `ControlUniqueId`: max = 267 → use 268, 269, 270, ... for new controls

## Task 1: EditScreen1 — Add ProjectTier_DataCard1 to Controls/65.json

The form currently has 25 DataCards at `Y` positions 0–24. "Project Tier" must be inserted at `Y=1` (2nd slot). This requires:

2. Incrementing Y by 1 for all existing DataCards at Y ≥ 1 (Department through the last card)

**Step 1: Increment existing DataCard Y values ≥ 1 in Controls/65.json**

Run this Python script. It specifically targets the `InvariantScript` for `Y` properties that are direct children of `EditForm1` (values 1–24) — NOT the internal Y values of labels/inputs inside cards (those are things like `"10"`, `"DataCardKey10.Y + DataCardKey10.Height + 5"`, etc.).

```python
#!/usr/bin/env python3
import re

with open('/tmp/msapp_extracted/Controls/65.json', 'r') as f:
    content = f.read()

# The form DataCard Y ordering values are single-digit or double-digit integers 1-24.
# They appear as: "Property": "Y", ... "InvariantScript": "N"
# where N is 1..24 and the parent is EditForm1 (DataCard controls).
# We replace them in reverse order (24 down to 1) to avoid double-incrementing.
import json

data = json.loads(content)

def increment_form_card_y(controls_list):
    for ctrl in controls_list:
        if ctrl.get('Parent') == 'EditForm1' and ctrl.get('Template', {}).get('Id') == 'http://microsoft.com/appmagic/card':
            for rule in ctrl.get('Rules', []):
                if rule.get('Property') == 'Y':
                    old_val = rule['InvariantScript']
                    try:
                        n = int(old_val)
                        if n >= 1:
                            rule['InvariantScript'] = str(n + 1)
                    except ValueError:
                        pass

# Walk all controls in the TopParent.Children array
top = data['TopParent']
def walk(control):
    increment_form_card_y(control.get('Children', []))
    for child in control.get('Children', []):
        walk(child)

walk(top)

with open('/tmp/msapp_extracted/Controls/65.json', 'w') as f:
    json.dump(data, f, indent=2, ensure_ascii=False)

print("Done incrementing DataCard Y values in Controls/65.json")
```

Run: `python3 /tmp/increment_datacards.py`

import re

# The DataCard ordering Y values appear as exactly:
#   "                  Y: =N\n"  (10 spaces + "Y: =N")
# where N is 1..24. Internal control Y values use different indentation or expressions.
# DataCard-level properties are indented 18 spaces (6 levels of 2-space indent + "Y: =").

def increment_datacards(text):
    # Match DataCard-level Y lines: 18 spaces indent, Y: =N (integer 1-24)

        old = '                  Y: =' + str(n) + '\n'
        new = '                  Y: =' + str(n + 1) + '\n'
        text = text.replace(old, new)
    return text
